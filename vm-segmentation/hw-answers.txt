4. Assume we want to generate a problem where roughly 90% of the
randomly-generated virtual addresses are valid (not segmentation
violations). How should you configure the simulator to do so?
Which parameters are important to getting this outcome?

Ans: Crux: Choose valid segment addresses that are 90% of the valid address space. 
So if address space is 100, 90 should be the valid segment size. this can be 45 for each segment. 

len0 and len1 (segment limits): choose so (len0+len1)/asize ≈ 0.90.
pick asize = 100, len0 = 45, len1 = 45 → (45+45)/100 = 0.90
ensure psize large enough for random base placement (psize > 200); use 4096 here

python segmentation.py -a 100 -p 4096 -n 5000 --l0 45 --l1 45 -s 42 -c

5. Can you run the simulator such that no virtual addresses are valid?
How?

Ans: Set both limit registers to 0. No VA will be valid since the valid segment size is set to 0.
